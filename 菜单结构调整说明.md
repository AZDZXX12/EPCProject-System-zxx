# EPC系统菜单结构调整说明

## 📋 调整概述

本次优化对EPC系统菜单进行了重新规划，主要目的是：
1. 分离"选型"和"采购"两个不同的业务流程
2. 优化菜单顺序，更符合业务逻辑
3. 优化设备选型系统页面布局

## 🎯 功能定位差异

### 设备采购管理 (P)

**定位**: 商务采购流程管理

**核心功能**:
- 设备采购清单管理
- 厂家信息管理
- 报价管理
- 交货期追踪
- 采购状态管理（待询价→已报价→已下单→已到货）
- 采购总额统计

**使用场景**:
- 采购部门日常工作
- 厂家询价和比价
- 合同签订和跟踪
- 到货验收记录

**数据来源**:
- 从设备选型系统导入技术清单
- 手动录入采购信息
- Excel批量导入

---

### 设备选型系统

**定位**: 技术参数选型工具

**核心功能**:
- 设备技术参数计算
- 选型表格填写
- 选型结果导出
- 多工作表管理
- 公式计算支持

**使用场景**:
- 工程师技术选型
- 参数计算和校核
- 选型方案对比
- 技术清单生成

**数据流向**:
- 导出清单到采购管理
- 生成技术文档

## 🔗 两个系统的协同关系

### 业务流程
```
设备选型系统 → 技术参数确定 → 生成选型清单
         ↓
   导入按钮
         ↓
设备采购管理 → 补充商务信息 → 询价比价 → 下单采购
         ↓
   状态追踪
         ↓
   到货验收
```

```

         │ 导出/导入
         ↓
┌─────────────────┐
│  设备采购管理   │
│  (商务层面)     │
│ +制造厂家       │

│ +报价信息       │
│ +交货期         │

└─────────────────┘
```

## 📍 菜单位置调整原因

### 为什么把选型系统移到人员管理上方？

1. **业务逻辑**:
   - 采购管理在P阶段（EPC中的P）
   - 选型系统是技术支持工具
   - 人员管理、文档库、报表属于通用支持模块
   - 选型系统放在支持模块区域更合理

2. **使用频率**:
   - 选型系统：项目前期频繁使用，后期较少
   - 采购管理：贯穿项目全程
   - 人员管理：日常管理功能
   - 分开放置，避免混淆

3. **用户体验**:
   - 采购人员主要使用"采购管理"
   - 工程师主要使用"选型系统"
   - 两个系统分开，各司其职
   - 减少误操作

## 🚀 使用建议

### 典型工作流

#### 场景1: 项目启动阶段
```
1. 工程师登录系统
2. 进入"设备选型系统"
3. 根据工艺要求进行设备选型
4. 填写选型表格（参数计算）
5. 完成选型，生成清单

6. 采购人员登录系统
7. 进入"设备采购管理"
8. 点击"从选型系统导入"
9. 获得技术清单
10. 补充厂家和报价信息
```

#### 场景2: 日常采购工作
```
1. 采购人员进入"设备采购管理"
2. 查看待询价设备清单
3. 联系厂家询价
4. 录入报价信息
5. 比价后确定供应商
6. 更新状态为"已下单"
7. 跟踪交货期
8. 到货后更新状态为"已到货"
```

#### 场景3: 紧急补充设备
```
1. 发现需要补充设备
2. 工程师先在"选型系统"中选型
3. 或直接在"采购管理"中手动添加
4. 进行采购流程
```

## ⚠️ 注意事项

### 1. 路由变化
- **旧路由**: `/selection` → 指向选型采购（已废弃）
- **新路由**: 
  - `/procurement` → 设备采购管理
  - `/selection` → 设备选型系统

### 2. 权限设置
- 采购管理：采购人员、项目经理
- 选型系统：工程师、技术人员、采购人员

### 3. 数据隔离
- 两个系统数据相对独立
- 通过"导入"功能实现数据传递
- 各自可独立使用
